# Kanso UI v0.0.0 — Initial Release

**Release Date:** June 9, 2026

---

## 🎉 Welcome to Kanso UI

Kanso UI is a thoughtfully designed, copy-paste React component library inspired by Japanese minimalism. This is the inaugural release that establishes the foundation for modern, accessible UI components.

**Version**: `0.0.0`  
**Status**: Alpha (Foundation Release)

---

## ✨ What's Included

### 📚 Documentation Infrastructure

- **Full-featured docs site** at `/docs` with elegant sidebar navigation
- **Dynamic component pages** that auto-generate from the registry
- **Live interactive previews** for every component
- **Copy-paste workflow** — browse, preview, copy source code
- **Installation guides** with step-by-step instructions
- **Props documentation** with searchable filters
- **Mobile-responsive** docs with drawer navigation
- **Dark mode support** throughout the documentation platform

### 🎨 Kanso-Branded Components (3 Initial)

1. **Magnetic Button** (`magnetic-button`)
   - Button with cursor-following spring animation
   - Customizable magnetic strength
   - Uses Framer Motion for smooth physics
   - Keyboard accessible with focus indicators

2. **Shimmer Border** (`shimmer-border`)
   - Container with animated shimmer effect
   - Pure CSS animation for performance
   - Customizable color, speed, and radius
   - Subtle premium aesthetic

3. **Text Reveal** (`text-reveal`)
   - Character-by-character reveal animation
   - Triggers on scroll into view
   - Staggered blur effect for elegance
   - Server-renderable with Framer Motion

### 🏗️ Registry System

- **Central component registry** (`lib/registry.ts`) — single source of truth
- **Auto-generating docs** — add registry entry → page auto-generates
- **Component metadata** with props, dependencies, and tags
- **API endpoint** (`GET /api/registry/[name]`) for future CLI tooling
- **JSON source delivery** for copy-paste workflow

## 🎨 Design Philosophy

**Kanso (簡素)** — Japanese principle of minimalism and simplicity

Every component follows these principles:

1. ✅ **Simplicity over cleverness** — fewer lines, fewer abstractions
2. ✅ **Readability over abstraction** — code that's easy to understand
3. ✅ **Composition over inheritance** — prefer children/slots over props
4. ✅ **Accessibility over visual tricks** — keyboard nav, ARIA, semantic HTML
5. ✅ **Performance over unnecessary features** — optimized and lean

## 🚀 Quick Start

### Installation

```bash
# Clone repository
git clone https://github.com/Nagraj-13/Kanso-UI.git
cd kanso-ui

# Install dependencies
pnpm install

# Start development server
pnpm dev
```

### View Documentation

- Landing page: `http://localhost:3000`
- Component docs: `http://localhost:3000/docs`
- Browse all 58 components (3 Kanso + 55 primitives)

### Add Your First Component

1. Create `components/kanso/my-component.tsx`
2. Register in `lib/registry.ts`
3. Add demo in `components/docs/component-demos.tsx`
4. Docs page auto-generates at `/docs/components/my-component`

See **CLAUDE.md** for detailed step-by-step examples.

## 🎓 Philosophy

Kanso UI is positioned alongside **shadcn/ui**, **Magic UI**, and **Aceternity UI**, but with a stronger focus on:

- **Minimalism** — neutral palette, generous whitespace
- **Accessibility** — keyboard navigation, screen reader support
- **Developer Experience** — type-safe, well-documented, easy to extend
- **Simplicity** — smaller API surface, sensible defaults
- **Performance** — server-renderable, CSS animations where possible
